**Copilot Studio**は、Microsoftが提供する**AIエージェント開発ツール**で、企業や組織が独自のCopilot（AIアシスタント）を簡単に作成・カスタマイズできるプラットフォームです。

### 🌟 主な特徴
- **ローコード開発**：プログラミングの知識がなくても、ドラッグ＆ドロップでエージェントを構築できます。
- **自然言語での指示**：AIに「〇〇をして」と話しかけるだけで、複雑な処理を自動化できます。
- **Microsoft 365との連携**：Teams、Outlook、SharePointなどと統合可能。
- **複数チャネル対応**：Webサイト、SNS、社内アプリなどにエージェントを展開できます。
- **高度なカスタマイズ**：ナレッジベース、トピック、アクション、ワークフローなどを細かく設定可能。

🧠 **Azure MCP Server（プレビュー）**は、AIエージェントや他のクライアントが自然言語でAzureリソースと対話できるようにするための**オープンプロトコル**に基づくサーバーです。

### 🔍 主なポイント
- **MCP（Model Context Protocol）**は、外部ツールやメモリ、コンテキストとのやり取りを安全かつ構造化された方法で管理するプロトコル。
- Azure MCP Serverは、GitHub Copilotエージェント、OpenAI Agents SDK、Semantic Kernelなどのクライアントと連携可能。
- **Entra ID**を使った認証に対応し、Azureのベストプラクティスに準拠。
- Azure CLIやAzure Developer CLI（azd）などのツールをサポート。
- MCPクライアントは、自然言語を使ってAzureリソースにアクセス・操作可能。

### 🛠 MCPの構成要素
| コンポーネント | 役割                                                 |
| -------------- | ---------------------------------------------------- |
| ホスト         | MCPクライアントと接続し、データを提供するアプリ      |
| クライアント   | MCPサーバーに接続し、データ取得や操作を行う          |
| サーバー       | 対話やアクション実行のためのツールやプロンプトを提供 |

### 💡 利用例
- VS CodeのGitHub CopilotエージェントがMCPクライアントとして機能し、Azureリソースに対して自然言語で操作。
- 独自のMCPクライアントをホストして、カスタムアプリケーションからAzureに接続することも可能。

より詳しく知りたい場合は、[公式ドキュメント](https://learn.microsoft.com/ja-jp/azure/developer/azure-mcp-server/overview)をご覧ください。興味があれば、どんなユースケースに活用できそうか一緒に考えてみましょうか？
</code>
